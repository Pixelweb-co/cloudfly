# Meta Ads API Documentation

## Overview
This document provides a comprehensive reference for the **Meta Ads** integration exposed by the `marketing_agent` service. It describes all six endpoints, request/response schemas, authentication, error handling, CLI usage, and configuration.

---

## Endpoints
| Method | Path | Description |
|--------|------|-------------|
| `POST` | `/meta/ads/create` | Create a new Meta Ad campaign. |
| `GET`  | `/meta/ads/{ad_id}` | Retrieve details of a specific ad. |
| `PATCH`| `/meta/ads/{ad_id}` | Update an existing ad (budget, status, etc.). |
| `DELETE`| `/meta/ads/{ad_id}` | Delete an ad campaign. |
| `GET`  | `/meta/ads` | List all ads with optional filters. |
| `POST` | `/meta/ads/{ad_id}/insights` | Pull insights/report for a given ad.

## Request Schemas
### Create Ad (`POST /meta/ads/create`)
```json
{
  "name": "string",
  "objective": "string",
  "status": "PAUSED | ACTIVE",
  "budget": {
    "amount": "number",
    "currency": "USD | EUR | ..."
  },
  "targeting": {
    "geo_locations": ["US", "CA"],
    "age_min": 18,
    "age_max": 65,
    "interests": ["technology", "sports"]
  }
}
```
*All fields are required unless marked optional.*

### Update Ad (`PATCH /meta/ads/{ad_id}`)
```json
{
  "status": "PAUSED | ACTIVE",
  "budget": {
    "amount": "number"
  }
}
```

## Response Schemas
### Successful Create / Update
```json
{
  "ad_id": "string",
  "status": "ACTIVE",
  "created_at": "2024-01-01T12:00:00Z",
  "message": "Ad created successfully."
}
```

### Error Response
```json
{
  "error": {
    "code": "string",
    "message": "Detailed error description",
    "details": {}
  }
}
```
Common error codes:
- `AUTH_ERROR` – Invalid or expired Meta access token.
- `VALIDATION_ERROR` – Request payload does not match schema.
- `RATE_LIMIT` – Too many requests, retry after `Retry-After` header.
- `INTERNAL_ERROR` – Unexpected server error.

## Authentication
All requests must include the HTTP header:
```
Authorization: Bearer <META_ACCESS_TOKEN>
```
The token is read from the environment variable `META_ACCESS_TOKEN` (see **Configuration**).

## CLI Usage (marketing_agent/main.py)
```bash
# Generate a new ad from a JSON template
python main.py --generate-ad template.json --meta-ads-budget 1000

# Directly create an ad via CLI
python main.py --create-meta-ads "{...json payload...}"
```
| Flag | Description | Default |
|------|-------------|---------|
| `--generate-ad` | Path to a JSON template used to generate an ad. | – |
| `--create-meta-ads` | Raw JSON payload to send to the create endpoint. | – |
| `--meta-ads-budget` | Budget amount (overrides template value). | `500` |

## Error Handling & Retry Strategy
1. **Authentication errors** – Refresh token manually and restart the service.
2. **Rate limiting** – Respect `Retry-After` header; exponential back‑off (1s, 2s, 4s…).
3. **Transient network errors** – Retry up to 3 times with jitter.
4. **Validation errors** – Return detailed message; fix payload before retry.
