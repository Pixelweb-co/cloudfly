# AGENTE_DEV_Architecture_ai_scrum_team.md

## Overview
The **ai_scrum_team** micro‑service is a FastAPI application that provides AI‑assisted Scrum functionalities such as ticket hand‑off routing. It runs in a Docker container and is exposed through **Traefik** under the path `/ia_scrum_team`.

## Deployment Diagram
```mermaid
graph LR
    Internet --> Traefik[Traefik (websecure)]
    Traefik --> IA[ai‑scrum‑team (FastAPI, 8000)]
    IA --> DB[(External DB – credentials via .env)]
    IA --> ModelCache[Docker volume: model_cache]
    Traefik --> Portainer[Portainer UI 9000]
```
*All traffic to the service is routed via Traefik, which handles TLS termination and path‑based routing.*

## Service Components
- **Dockerfile** – Multi‑stage build, non‑root user, health‑check on `/health`.
- **docker‑compose.yml** – Defines the container, ports, environment, and volume for model cache.
- **FastAPI Endpoints**
  - `GET /health` – Liveness probe.
  - `GET /ready` – (optional) readiness probe checking DB and cache.
  - `POST /handoff` – Core AI hand‑off logic (see API contract below).

## Security Recommendations
- Store secrets via Docker secrets, not in `.env`.
- Run container as non‑root user (`appuser`).
- Limit resources in compose (`cpu: 0.5`, `memory: 512M`).
- Expose only required ports (8000).
